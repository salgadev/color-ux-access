> Project context for AI agents. Detailed docs at `docs/`.

## What This Project Is

A Gradio app that tests any webpage screenshot for colorblind accessibility issues.

**Pipeline:** screenshot ‚Üí 10-type CVD simulation (DaltonLens) ‚Üí VLM WCAG 2.1 audit ‚Üí report.

- **Entry point:** `app.py` (root) ‚Äî single file, Gradio 5/6 compatible.
- **VLM:** MiniCPM-v-4.6 (~4B params) via Modal endpoint (`MODAL_INFERENCE_URL` env var).
- **CVD:** 10 types ‚Äî deuteranopia, protanopia, tritanopia, deuteranomaly, protanomaly, tritanomaly, severe_deuteranopia, severe_protanopia, achromatopsia, achromatomaly.

## Key Conventions

### Environment

- **Local dev:** `uv sync --python 3.12` ‚Äî all deps via `pyproject.toml` + `uv.lock`.
- **HF Spaces:** `requirements.txt` is generated by `uv export --format requirements-txt`, never hand-edited.
- **Never source `.venv/activate`** ‚Äî use `uv run` instead.

### Dependency changes

```bash
uv add some-package          # or edit pyproject.toml manually
uv sync                      # updates uv.lock
uv export --format requirements-txt --output-file requirements.txt --no-editable --no-emit-project --no-hashes
git commit pyproject.toml uv.lock requirements.txt
```

### Testing

- **HF Space constraint:** containers cannot run a browser. Tests use source inspection only.
- **Image input:** The app expects a screenshot image to be uploaded directly by the user. No URL capture or browser automation is needed.
- **ALWAYS use `uv run pytest`** ‚Äî ensures the project venv is used, NOT the global Python. Running `python -m pytest` directly will use Hermes Agent's global environment and may cause dependency conflicts.
- **No VLM calls in tests** ‚Äî use `mock_vlm_factory` from `conftest.py`.

### File structure

```
app.py              # Gradio app (root ‚Äî HF Space entrypoint)
color_ux_access/    # package: cvd_sim, capture, modal_app
vlm/                # VLM inference modules
tests/              # pytest, source inspection only
docs/               # detailed documentation
pyproject.toml      # source of truth (local dev / CI)
requirements.txt    # generated (Spaces deployment)
```

### Gradio version handling

- HF Spaces SDK installs Gradio 5.x; local dev uses 6.x.
- `app.py` handles this at runtime via `_is_gradio6` check.
- Theme/CSS go to `Blocks()` constructor (Gradio 5) or `launch()` (Gradio 6).

### VLM backend

The app uses `_call_minicpm_endpoint` in `app.py` which POSTs to the
Modal inference URL configured via `MODAL_INFERENCE_URL`.
The model served on Modal is MiniCPM-v-4.6 (~4B params).
There is no swappable-model dict ‚Äî the endpoint model is configured
on the Modal side.

## Important Rules

1. Do not install packages globally or run `pip install` outside `uv` ‚Äî pollutes Hermes Agent's environment.
2. HF Spaces does not read `pyproject.toml` ‚Äî always regenerate `requirements.txt` after dependency changes.
3. The app expects a screenshot image to be uploaded directly ‚Äî no URL capture or browser automation is needed.
4. Tests must not call the VLM ‚Äî mock it with `mock_vlm_factory` from `conftest`.

### Pre-commit Hooks

A local pre-commit hook (`block-root-fix-scripts` in `.pre-commit-config.yaml`) blocks commits that stage Python files matching `*fix*.py` or `apply_*.py` at the repository root. This prevents one-off fix scripts from being committed to the root directory.

- **Allowed:** `scripts/fix_issue.py`, `tests/test_fix.py`, `normal_script.py`
- **Blocked:** `fix_issue.py`, `apply_dummy.py`, `fix_v2.py` (at root)

To temporarily bypass: `git commit --no-verify` (use sparingly).
